# 手取り計算ロジック・顧客説明用メモ

## 📝 手取り計算ロジック（所得税・住民税の累進課税）

### 1. 段階的税率の採用理由

所得税・住民税は超過累進課税制度を採用しており、年収が上がるにつれて税金が引かれる割合（実効税率）が高くなります。

### 2. 年収帯ごとの税率提案

| 年収帯 | 税率候補 | 理由 |
|--------|---------|------|
| 〜400万円 | 15.0% | 年収の低い層は基礎控除や配偶者控除などの影響が大きく、実効税率も低くなるため |
| 400万〜800万円 | 20.0% | 所得税率が高くなる区分に入り始め、実効税率が上昇するため |
| 800万〜1,200万円 | 25.0% | 所得税率がさらに高まり、手取りに占める税金の割合が大きくなるため |
| 1,200万円超 | 30.0% | 所得税の最高税率区分に近づき、非常に高い実効税率となるため |

### 3. 採用理由（固定レートの優位性）

| 理由 | 詳細な説明 |
|------|-----------|
| 累進課税の反映 | 社会保険料が上限で頭打ちになるのに対し、税金は年収に応じて負担率が上がるという、日本の制度のリアリティを簡易的に表現するため、段階的な係数を採用しました |
| 計算の安定性 | 正確な税額計算は、扶養親族の数、医療費控除、生命保険料控除など、数百種類の要素で変動します。シミュレーションの安定性を優先し、控除を標準的なものと見なした概算の実効税率を採用しています |
| 顧客への視覚的訴求 | 年収帯によって手取り率が明確に下がることを示すことで、「保障額を確保しても、税金で手元に残る額はこれくらい」という将来の現実に即した感覚を顧客に訴求できます |

## 🗣️ 顧客説明用トーク（社会保険料率15.0%固定の根拠）

### 「社会保険料の計算と高年収層の実質負担率に関するご説明」

このシミュレーションで「手取り月収」の表示を切り替える際、社会保険料率は**一律で15.0%**として概算しています。これは、計算の複雑化を防ぎ、分かりやすい目安をお伝えするための措置ですが、特に年収の高いお客様にご理解いただきたい重要な点があります。

日本の社会保険料（厚生年金保険と健康保険が主な構成要素）には、**『上限（キャップ）』**が設定されています。所得税が年収に応じて青天井で税率が上がるのに対し、社会保険料は一定の金額で頭打ちになるという特性があるのです。

具体的には、厚生年金保険料は標準報酬月額が65万円（年収約780万円）を超えるとそれ以上は上がらず、健康保険料も同様に上限が設定されています。

このため、年収が780万円を超えて1,000万円、1,500万円と上がっていくにつれて、保険料の絶対額はほぼ固定されるため、**『年収全体に占める社会保険料の割合（実質的な負担率）』は徐々に下がっていきます**。

つまり、このシミュレーションで一律15.0%として表示しているのは、多くの年収帯で妥当な**『最大公約数的な目安』**であり、お客様の実際の社会保険料の実質的な負担率は、この15.0%よりも低くなっている可能性が高いということです。

弊社では、お客様が保障額を過小評価することのないよう、あえてこの水準で固定していますが、この点をご理解いただければ幸いです。

## 📊 実装されている計算ロジック（参考）

現在の実装では、以下の計算ロジックが使用されています：

- **社会保険料率**: 15.0%（固定、全国一律）
- **所得税・住民税率（年収帯別）**:
  - 年収 ≤ 400万円: 8.0% → 手取り係数 **0.770**
  - 400万円 < 年収 ≤ 800万円: 10.0% → 手取り係数 **0.750**
  - 800万円 < 年収 ≤ 1200万円: 10.0% → 手取り係数 **0.750**
  - 年収 > 1200万円: 15.0% → 手取り係数 **0.700**

**注意**: 上記の実装ロジックは、提案段階の税率（15.0%、20.0%、25.0%、30.0%）とは異なります。
